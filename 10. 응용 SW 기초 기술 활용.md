# 93. 운영체제의 개념

1. 운영체제 : 컴퓨터의 자원을 효율적으로 관리하며 사용자가 컴퓨터를 편리하고 효과적으로 사용할수 있도록 환경을 제공하는 여러 프로그램의 모임.
2. 목적, 성능 평가 기준(**처신 반사**)
   - **처**리 능력향상 : 일정 시간내 시스템이 처리하는 일의 양
   - **신**뢰도 향상 : 시스템이 주어진 문제를 정확하게 해결하려는 정도
   - **반**환 시간 단축 : 시스템에 작업한 의뢰를 처리 완료될 때까지 걸린시간
   - **사**용 가능도 향상 : 시스템을 사용할 필요가 있을 때 즉시 사용가능한 정도
3. 운영체제 주요 자원 관리
   - 프로세스 관리 : 프로세스 스케줄링 및 동기화 관리 담당
   - 기억장치 관리 : 프로세스에게 메모리 할당 및 회수 관리 담당
   - 주변장치 관리 : 입출력장치 스케줄링 및 전반적인 관리 담당
   - 파일 관리 : 파일의 생성과 삭제,변경,유지 등의 관리 담당

# 95. 운영체제의 기본 명령어

1. window 기본 명령어

   - **dir** : 현재 디렉터리의 파일목록 표시
     dir
     옵션

     - /p : 목록을 한 화면 단위로 표시
     - /w : 목록을 가로로 나열
     - /o : 지정한 정렬 방식으로 파일 목록표시
       - d : 날짜
       - e : 확장자
       - n : 파일이름
       - s : 파일크기
     - /s : 하위 디렉터리 정보까지
     - /a : 지정한 속성이 설정된 파일목록 표시

   - **copy** : 파일복사
     copy abc.txt gilbut : abc.txt파일을 gilbut디렉터리로 복사

   - **del** : 파일 삭제
     del abc.txt : abc.txt파일 삭제

   - **type** : 파일 내용 표시
     type abc.txt : abc.txt파일 내용 표시

   - **ren** : 파일 이름 변경
     ren abc.txt 123.txt : abc.txt파일 이름을 123.txt로 변경

   - **md** : 디렉터리 생성
     md gilbut : gilbut디렉터리 생성

   - **cd** : 디렉리의 위치 이동
     cd gilbut : gilbut디렉터리로 이동

   - **cls** : 화면 내용 지움
     cls

   - **attrib** : 파일 속성 변경

     속성 종류(속성 추가:+, 속성 취소: -)

     - r : 읽기
     - a : 저장

   - **find** : 파일에서 문자열찾기

   - **chkdsk** : 디스크 상태 점검
     chkdsk : 현재 드라이브의 상태 점검

   - **format** : 디스크 표면을 트랙과 섹터로 나누어 초기화
     format c: : c드라이브를 초기화

     move abc.txt gilbut : abc.txt파일을 gilbut디렉터리로 이동

# 96. 기억장치 관리 

1. 반입(fetch)전략 : 데이터를 주기억장치로 언제 적재할 것인지 결정하는것
   - 요구반입 : 참조를 요구할때 적재
   - 예상반입 : 예상하고 미리 적재
2. 배치(placement)전략: 데이터를 주기억장치 어디에 위치시킬것인지
   - 최초 적합 : 첫번째 분할 영역에 배치시키는 방법
   - 최적 적합 : 단편화를 가장 작게 남기는 분할 영역
   - 최악 적합 : 단편화를 가장 많이 남기는 분할 영역
3. 교체(replacement)전략
   페이지 교체 알고리즘(**FOLLNS**)
   - **OPT**(OPTimal replacement, 최적교체) : 앞으로 오래 사용하지 않을 페이지를 교체
   - **FIFO**(First In First Out) : 가장 먼저 들어와 가장 오래있었던 페이지 교체
   - **LRU**(Least Recently Used) : 최근에 가장 오랫동안 사용하지 않았던 페이지 교체
   - **LFU**(Least Frequently Used) : 사용 빈도가 가장 적은 페이지 교체 
   - **NUR**(Not Used Recently) : 최근에 사용하지 않은 페이지 교체, 참조 비트와 변형 비트 사용
   - **SCR**(Second Chance Replacement) : 가장 오랫동안 있었던 페이지 중 자주 사용되는 페이지가 교체되는것을 방지하기 위한 것
4. LINUX메모리 관리방법
   - memeinfo 명령어를 통해 메모리 상태 확인
   - 메모리가 부족한경우
     - Swapping기법 : 프로그램에 할당된 메모리의 일부를 보조기억장치에 저장하는 기법
       dd if=/dev/zero of=[파일명] bs=[용량] count=1024(반복횟수)
       follocate --l 1G /swapfile : follocate를 이용해 1Gbyte스왑파일 생성
     - min_free_kbytes커널파라미터
       /proc/sys/vm/min_free_kbytes=20480 : 가용 메모리 크기를 20480kbyte로 설정

# 97. 프로세스 관리

1. 프로세스 상태 전이

   ![image](https://user-images.githubusercontent.com/57162257/115147280-3eda5780-a095-11eb-8f54-d827b283232b.png)

   - 제출 : 작업 처리를 위해 사용자가 작업을 시스템에 제출한 상태
   - 접수 : 제출된 작업이 스풀 공간인 디스크의 할당 위치에 저장된 상태
   - 준비 : 프로세스가 프로세서를 할당받기 위해 기다리는 상태
     //접수 상태에서 준비 상태로의 전이는 Job스케줄러에 의해 수행
   - 실행 : 프로세스가 프로세서를 할당받아 실행되는 상태
     //실행중인 프로세스에 입출력처리가 필요하면 실행중인 프로세스는 대기 상태로 전이
     //준비상태에서 실행상태로의 전이는 CPU 스케줄러에 의해 수행
   - 대기 : 입출력 처리가 완료될떄까지 대기
   - 종료 : 프로세스의 실행이 끝나고 프로세스 할당이 해제된 상태
   - spooling : 입출력할 데이터를 직접 인출력장치에 보내지않고 나중에 한꺼번에 입출력하기 위해 디스크에 저장하는 과정
   - dispatch : 준비 상태에서 프로세서를 할당받아 실행 상태로 전이되는 과정

2. 프로세스 스케줄링 알고리즘

   - **선점형 스케줄링** : 실행중인 프로세스가 있는데 우선순위가 높은 프로세스가 있는경우 현재 실행중인 프로세스를 중지시키고 순위가 높은 프로세스 프로세서 할당
     - **라운드로빈**(RoundRobin) : 시간안에 완료하지 못하면 뒤로 보냄
     - **SRT**(Shortest Remaning Time first) : 가장 짧은 처리 우선 수행
     - **다단계 피드백 큐** : RR+FCFS, 우선순위 단계
     - **선점 우선순위** : 준비상태 큐에서 우선순이가 가장 높은 프로세스부터 먼저 CPU할당
   - **비선점형 스케줄링** : 실행중인 프로세스가 있다면 다른 프로세스는 CPU를 점유할수 없다.
     - **FCFS**(First Come First Service)=FIFO : 먼저들어온 순서대로 CPU할당
     - **SJF**(Shortest Job First) : 준비상태 큐에서 실행시간이 가장 짧은 프로세스에게 먼저 할당
     - **HRN**(Highest Response-ratio Next) : 응답률 순서, (대기시간+서비스 시간)/서비스 시간
     - **기한부** : 명시된 시간내에 수행
     - **우선순위** : 준비상태 큐에서 우선순위를 부여
   - **에이징(Aging)기법**
     //특정 프로세스의 우선순위가 너무 낮아 무한정 기다리게되는 경우, 기다린 시간에 비례하여 우선순위를 한단계씩 높여줘서 자원을 할당받을수 있게 해주는 기법.

3. 교착상태(Dead Lock)
   //둘 이상의 프로세스가 서로 다른 프로세스가 점유하고 있는 자원을 요구하며 무한정 기다리는 현상.

   - 교착 상태의 필요 충분조건(**상점 비환**)
     - **상**호 배제 : 한번에 한개의 프로세스만이 공유 자원을 사용할수 있어야한다
     - **점**유와 대기 : 하나의 자원을 점유하고 있으면서 다른 프로세스에 할당되어있는 자원을 점유하기위해 대기하는 프로세스가 있어야한다.
     - **비**선점 : 다른 프로세스에 할당된 자원은 자원사용이 끝나기전에는 할당받을수 없다.
     - **환**형 대기 : 공유 자원을 사용하기 위해 대기하는 프로세스가 원형으로 구성되어있는 경우.

4. 교착상태 해결방법(**예회발회**)

   - **예**방 기법: 교착 상태가 발생하지 않도록 사전에 시스템을 제어하는 방법,
     교착상태 발생의 네가지 조건 중에서 하나를 제거함으로써 수행.
   - **회**피 기법 : 교착상태가 발생하면 피해가는 방법(은행원 알고리즘)
   - **발**견 기법 : 교착 상태에 있는 프로세스와 자원을 발견
   - **회**복 기법 : 교착상태를 만든 프로세스 종료, 교착상태의 프로세스에 할당된 자원을 선점.

   - 데이터 **마**트 : 데이터 웨어하우스로 부터 특정 주제로 구축된 소규모 단일 주체 데이터 웨어하우스
   - 데이터 **마**이닝 : 데이터 웨어하우스에서 요구에 따라 유용하고 가능성있는 정보를 발견하기 위한 기법
   - **O**LAP(OnLine Analytical Processing) : 다차원으로 이루어진 데이터로부터 통계적 요약 정보를 분석하여 의사결정에 활용하는 방식
   - **O**LTP(OneLine Transaction Processing) : 네트워크상 여러 이용자가 실시간으로 데이터베이스 데이터를 갱신하거나 검색하는 등의 단위 작업을 처리하는 방식

# 101. 키의 개념 및 종류

1. 키(Key) : DB에서 조건에 만족하는 튜플을 찾거나 정렬할때 기준이 되는 속성.
2. 슈퍼키 : 모든 튜플에 대해 유일성은 만족하지만 최소성은 만족시키지 못함.
3. 후보키 : 튜플을 유일하게 식별하기 위한 속성의 집합, 유일성과 최소성 만족
4. 기본키 : 후보키 중에서 선정되 키, 중복값과 NULL값을 가질수 없다.
5. 대체키 : 후보키에서 기본키를 제외한 나머지.
6. 외래키 : 다른 릴레이션의 기본키를 참조하는 속성

# 102. 무결성

1. 무결성 : 데이터베이스에 저장된 값과 현실 세계의 실제값이 일치하는 정확성 의미.(**도개참사 널고 키관**)
   - **개**체 무결성 : 기본키는 NULL값과 중복값을 가질수 없다
   - **도**메인 무결성 : 속성값이 도메인에 속한값이어야한다
   - **참**조 무결성 : 외래키 값은 NULL이거나 참조 릴레이션의 기본키 값과 동일해야한다
   - **사**용자 정의 무결성 : 속성값들이 사용자가 정의한 제약조건에 만족해야한다.
   - **NULL** 무결성 : 특정 속성값이 NULL이 될수 없다
   - **고유** 무결성 : 특정 속성에 대해 각 튜플이 갖는 속성값은 달라야한다.
   - **키** 무결성 : 하나의 릴레이션에는 적어도 하나의 키가 존재해야한다.
   - **관계** 무결성 : 한 릴레이션과 다른 릴레이션의 튜플 사이의 관계에 대한 적절성 여부

# 106. TCP/IP

1. 프로토콜 : 서로 다른 기기들간의 데이터 교환을 원활하게 수행할수 있도록 표준화시켜 놓은 통신 규약
   - 프로토콜의 기본 요소(**의시구**)
     - **의**미 : 정보 전송을 위한 협조 사항과 오류 관리를 위한 제어 정보 규정
     - **시**간 : 통신 속도, 메시지의 순서 제어 등을 규정
     - **구**문 : 전송하고자 하는 데이터 형식, 부호화, 신호 레벨 등을 규정
2. TCP/IP의 구조
   - 응용 계층(응용,표현,세션 계층)
     //응용 프로그램 간의 데이터 송수신을 제공
     //TELNET, FTP, SMTP, DNS, HTTP
   - 전송 계층(전송 계층)
     //호스트들 간의 신뢰성 있는 통신
     //TCP, UDP
   - 인터넷 계층(네트워크 계층)
     //데이터 전송을 위한 주소 지정, 경로 설정
     //IP, ICMP, IGMP, ARP, RARP
   - 네트워크 엑세스 계층(데이터 링크, 물리 계층)
     //실제 데이터(프레임)를 송수신하는 역할을 한다.
     //Ethernet
3. TCP/IP 응용 계층 프로토콜
   - FTP(File Transfer Protocol) : 컴퓨터와 인터넷 사이에서 **파일을 주고 받을수** 있도록 하는 원격 파일 전송 프로토콜
   - SMTP(Simple Mail Transfer Protocol) : **전자 우편 교환** 서비스
   - TELENET : **떨어져있는 컴퓨터에 접속**하여 자신의 컴퓨터 처럼 사용할수 있는 서비스
   - SNMP(Simple Network Management Protocol) : 네트워크 기기의 **네트워크 정보를 네트워크 관리 시스템에 보내는 서비스**
   - DNS(Domain Name System) : 도메인 네임을 **IP로 매핑**해주는 서비스
   - HTTP(Hyper Text Transfer Protocol) : www에서 **HTML문서를 송수신 하기 위한 표준 프로토콜**
4. TCP/IP 전송 계층 프로토콜
   - TCP
     - 양방향 연결형 서비스 제공
     - 가상 회선 연결 형태의 서비스 제공
     - 스트림 위주의 전달(패킷단위)
     - 신뢰성 있는 메시지 전달
   - UDP
     - 비연결형 서비스 제공
     - 빠른속도 와 많은 사용자에게 전송할경우
     - 실시간 전송에 유리하며 신뢰성보다는 속도가 중요시되는 네트워크에 사용
   - RTCP(Real-Time Control Protocol)
     - RTP패킷의 전송품질을 제어하기 위한 제어 프로토콜
     - 하위 프로토콜은 데이터 패킷과 제어 패킷의 다중화를 제공
5. TCP/IP 인터넷 계층 프로토콜
   - IP(Internet Protocol) 
     - 전송할 데이터에 **주소 지정 및 경로 설정** 등의 기능
     - 비연결형인 데이터그램 방식을 사용하므로 신뢰성 보장되지않음
     - **패킷단위로 데이터 교환하는 통신 프로토콜**
   - ICMP(Internet Control Message Protocol)
     - IP와 조합하여 통신중에 발생하는 오류 처리와 전송 경로 변경들을 위한 **제어 메시지 관리 역할**
     - **IP패킷을 처리할때 오류발생시 알려주는 프로토콜**
   - IGMP(Internet Group Management Protocol)
     - 멀티캐스트를 지원하는 호스트나 라우터 사이에서 **멀티캐스트 그룹을 유지**하기 위해 사용
   - ARP(Address Resolution Protocol)
     - 호스트의 IP주소를 호스트와 연결된 네트워크 접속 장치의 **물리적 주소(MAC Address)로 바꿈**
   - RARP(Reverse Address Resolution Protocol)
     - ARP와 반대로 물리적 주소를 **IP주소로 변환**하는 기능
     - **MAC주소는 알지만 IP주소를 모를때 서버에 IP주소를 요청하는 프로토콜**

   - **R**IP(Routing Information Protocol) : 최초 라우팅 프로토콜, 라우팅 루프 발생, 소규모 네트워크
   - **I**GRP(Interior Gateway Routing Protocol) : RIP업그레이드, 중규모 네트워크
